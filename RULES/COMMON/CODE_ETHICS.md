# 🧭 코드 윤리 및 검증 지침 (Code Ethics)

## 1. 명명 규칙
- 클래스/함수/변수 식별자는 **영어**를 사용합니다.
- 의미가 불명확한 축약어(`tmp`, `val2`, `data1`) 사용을 지양합니다.

## 2. 주석 규칙
- 주석은 **한국어**로 작성하되, 코드만으로 명확한 경우 불필요한 주석을 추가하지 않습니다.
- 의도/제약/예외 처리 근거처럼 코드만으로 드러나지 않는 정보를 우선 기록합니다.

## 3. 변경 원칙
- 기존 동작을 바꾸는 경우 영향 범위를 문서 또는 커밋 메시지에 명확히 남깁니다.
- 추측 기반 수정 대신 재현 가능한 로그/증거를 기준으로 수정합니다.

## 4. 검증 원칙
- 변경 후 최소 1회 이상 빌드 또는 테스트로 결과를 검증합니다.
- 검증하지 못한 항목은 "미검증"으로 명시하고 리스크를 기록합니다.
